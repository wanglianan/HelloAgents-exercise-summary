# 第五天：工程化的提示词

**Hello-Agents结构化提示词章节**  
源于：第三章 大语言模型基础.md -> 3.2 与大语言模型交互

**核心观点：** 提示词 = 给大模型写 API 文档，吃透角色 + 任务 + 输出契约三核心  
**五大技术：** 零样本、少样本、思维链、角色扮演、格式强制控制五大基础能力
**工程化迭代：** 写-测-评-改-版本管理流程，用promptfoo 做提示词效果对比测试

**提示词通用模板**

	
	【系统设定｜System】
	角色（Role）：{{角色身份+专业背景+能力边界+语气风格}}
	定位（Position）：{{专家/助手/顾问/工程师，一句话定义价值}}
	约束（Constraints）：
	- 语言：{{中文/英文/中英双语，正式/口语/简洁/专业}}
	- 长度：{{最大字数/段落数/要点数}}
	- 格式：{{Markdown/JSON/表格/列表/纯文本}}
	- 禁止：{{禁止幻觉、禁止编造数据、禁止敏感内容、禁止废话}}
	- 必须：{{必须基于事实、必须逻辑清晰、必须分点、必须给出结论}}
	
	【上下文｜Context】
	背景（Background）：{{业务场景、用户画像、历史信息、输入数据}}
	目标（Goal）：{{一句话明确要解决什么问题、达成什么结果}}
	输入（Input）：{{用户问题/待处理文本/数据/需求原文}}
	
	【任务指令｜Task】
	指令（Instruction）：{{精确动作动词+任务细节+步骤要求}}
	步骤（Steps）：
	1. {{第一步：理解/分析/拆解}}
	2. {{第二步：推理/计算/生成}}
	3. {{第三步：总结/输出/格式化}}
	
	【输出契约｜Output Contract】
	结构（Structure）：
	- 标题：{{必填}}
	- 核心结论：{{1句话}}
	- 详细分析：{{分点，≤5条}}
	- 风险/限制：{{如有}}
	- 建议/下一步：{{如有}}
	格式（Format）：{{严格Markdown，代码块用```，表格用|}}
	质量标准（Quality）：{{无幻觉、逻辑自洽、可验证、可执行}}
	
	【示例｜Example（Few-Shot，可选但强烈推荐）】
	输入：{{示例输入}}
	输出：{{符合格式的高质量输出}}
	
	【评估与迭代｜Evaluation】
	成功标准（Success Criteria）：
	- {{可量化指标1}}
	- {{可量化指标2}}
	- {{可量化指标3}}
