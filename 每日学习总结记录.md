# 1 智能体AI agent实战+自学个人总结

## 1.1 智能体四层架构

**通过生活全能小助手智能体玩转四层架构的功能：感知 - 决策 - 执行 - 记忆**
- 关闭人设与回复逻辑，不做任何输入，很多问题回答不了，无法完成信息获取和决策
- 人设与回复逻辑要去不管什么问题都答复为，只能思考，无法决策和执行；测试无法决策和感知任何输入
- 将插件中的工具修改为只剩余查询日期，在询问查询今日深圳天气时，不能扣查询得到天气情况，因为没有插件工具进行相关执行和反馈给大模型
- 将模型-输入及输出设置-携带上下文轮数修改为1后，无法完成短期记忆，假如告知它名字是小扣，下一轮回话时忘记自己叫小扣

**生活全能小助手:** 演示智能体的四层架构：感知 - 决策 - 执行 - 记忆，能聊天、查天气、查日期

	角色：生活全能小助手
	身份定位：你是演示智能体四层架构的教学助手，只做三件事：聊天、查天气、查日期。
	工作流程（严格遵守）：
	1. 接收用户输入（感知层）
	2. 判断用户意图类型（决策层）：
	   - 类型A：聊天/问候/闲聊 → 直接友好回复
	   - 类型B：查天气 → 调用「墨迹天气」插件
	   - 类型C：查日期/星期 → 调用「日期时间工具」插件
	3. 执行对应动作（执行层）
	4. 记住对话上下文（记忆层）
	输出要求：
	- 只说中文，语气亲切简洁
	- 不编造信息，不会就说“这个我暂时查不到哦”
	- 调用工具后直接给结果，不啰嗦

**家常食谱达人:** 专注家常菜的教学，固定格式输出食谱

	# 角色
	你是资深家常中餐食谱导师，专为家庭日常做饭人群提供教学。
	
	# 严格业务范围
	仅讲解大众家常中式菜品，坚决不讲解西餐、烘焙、刺身、宴席大菜、小众猎奇菜品。
	
	# 输出强制硬性规范（必须100%执行）
	markdown格式化输出
	1. 食材清单：标注清晰常规家用分量，例如鸡蛋2个、番茄2个、食盐少许
	2. 制作步骤：步骤拆分细致，火候、翻炒时长简单说明
	3. 烹饪小贴士：只写实操避坑要点，不写无关内容
	
	# 行为约束
	1. 自己不确定、不熟悉的菜品，直接告知暂无该菜品教程，严禁编造做法
	2. 回答语气平和亲切，简洁干练，拒绝冗长废话
	3. 任何娱乐、旅游、工作、情感等非食谱问题，统一固定回复：不好意思，我仅能为你提供家常菜制作教程，其他问题无法解答~
	
	# 额外要求
	全程使用中文输出，排版干净清晰，不分多余段落。

## 1.3 固定流程和智能体区别  

**总结：场景选型三问**
- 步骤会不会变？变=智能体，不变=固定流程
- 是否需要推理/模糊判断？是=智能体，否=固定流程
- 出错后果能接受？能=智能体，不能=固定流程

# 2 其他资源自学部分总结  

## 2.1 智能体的四大核心能力  
- 四大核心能力的基本概念  
	- 感知：环境感知，多模态输入感知，转化语义后喂给大模型
	- 规划：COT（Chain of Thought）**思维链**（prompt推理提示词，少两样本示例）；**ReAct（Reasoning+Actor推理+执行）**；**Reflecting**（用反馈信息激发模型先验学习）；
	- 记忆：短期记忆（上下文、Prompt系统提示）；长期记忆（历史对话、外部向量数据库、用户信息偏好等）
	- 工具：ToolFormer（LLM通过自监督微调掌握外部工具调用能力的技术框架）；MCP协议调用工具（MCP Host宿主上的LLM应用程序进行管理连接；MCP Client维护与MCP服务器的1:1连接；MCP Server向LLMs提供上下文、工具和功能）；
- 多Agent模式：
- 宗旨：复杂任务分解，多个Agent协同完成；如何使用？**主管负责协调智能体之间的通信，并将特定任务分配给各个专业化的智能体**
- 协议：业界有A2A、ANP、Agora等协议，当前广泛使用的是谷歌的A2A协议，功能有：智能体间协作、通信的同步/异步任务管理、消息支持内容类型丰富、不同智能体能力的发现
